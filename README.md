# 💻 Formação Profissional em Web Designer (160h) — SENAC São Paulo

Este repositório centraliza todos os **códigos estruturados, exercícios práticos, resoluções de situações-problema e projetos** desenvolvidos ao longo das aulas do curso de Qualificação Profissional em Web Designer. 

O objetivo deste espaço é servir como um portfólio técnico vivo, documentando a evolução do aprendizado desde os conceitos de usabilidade e arquitetura de informação até o desenvolvimento e a publicação de aplicações web responsivas e acessíveis.

---

## 🎯 Proposta Pedagógica do Curso

O curso é estruturado sob o **Modelo Pedagógico SENAC**, centrado no desenvolvimento de competências profissionais baseadas no CHA (Conhecimentos, Habilidades e Atitudes). Toda a dinâmica em laboratório baseia-se na **aprendizagem por ação**, onde os desafios técnicos simulam cenários e demandas reais do mercado de trabalho.

A formação divide-se em 4 Unidades Curriculares (UCs) complementares, descritas em detalhe abaixo:

---

## 📚 Estrutura das Unidades Curriculares (UC) e Conteúdos

### 🗓️ UC1: Elaborar Projetos de Aplicações para Web
* **Foco:** Concepção, design de interface, arquitetura de informação e usabilidade.
* **O que foi aprendido e praticado:**
  * **UX/UI Design & Usabilidade:** Aplicação prática dos conceitos baseados nas diretrizes de Steve Krug ("Não Me Faça Pensar"). Princípios de legibilidade, hierarquia visual e design centrado no utilizador.
  * **Arquitetura de Informação:** Criação de mapas de site (*sitemaps*), fluxogramas de navegação e taxonomia de conteúdos.
  * **Prototipagem:** Transição do rascunho em papel (esboço de baixa fidelidade) para a estruturação de *wireframes* lógicos para desktop e dispositivos móveis.

### 🗓️ UC2: Desenvolver Aplicações para Websites
* **Foco:** Programação *front-end* nativa através da escrita de código limpo, semântico e padronizado.
* **O que foi aprendido e praticado:**
  * **HTML5 Semântico:** Estruturação lógica de layouts utilizando tags estruturais exclusivas (`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`) para eliminar o uso abusivo de `<div>` e otimizar o SEO (indexação em motores de busca).
  * **Acessibilidade Digital (WCAG):** Implementação de técnicas de acessibilidade, como o uso obrigatório e descritivo do atributo `alt` em imagens para leitores de ecrã e navegação por teclado.
  * **Validação Sintática W3C:** Submissão e correção de todos os códigos desenvolvidos através do validador oficial da *W3C (Markup Validation Service)* para atingir conformidade técnica internacional.
  * **CSS3 Avançado & Layouts Responsivos:**
    * Domínio do *Box Model* (margens, bordas, preenchimentos e dimensões).
    * Sistemas de posicionamento (`position: relative`, `position: absolute`, `static` e `fixed`).
    * Construção de layouts flexíveis, fluidos e modernos utilizando **CSS Flexbox**.
    * Aplicação de regras de media queries para garantir a total adaptabilidade do site a qualquer tamanho de ecrã (Design Responsivo).
  * **JavaScript Vanilla (Nativo):** Lógica de programação aplicada à interação web (variáveis, estruturas de decisão, funções) com foco direto na **Manipulação do DOM**. Desenvolvimento de componentes dinâmicos de interface (como menus hambúrguer para mobile, janelas modais e interações de botões).
  * *Nota institucional: Proibição estrita do uso de bibliotecas externas ou frameworks (como Bootstrap, Tailwind ou React), garantindo o domínio completo do código nativo.*

### 🗓️ UC3: Publicar Aplicações Web
* **Foco:** Preparação, otimização e distribuição de sistemas web para ambientes de produção.
* **O que foi aprendido e praticado:**
  * **Otimização de Mídias:** Conversão e compressão de imagens para formatos modernos de alta performance (como WebP) com foco em velocidade de carregamento.
  * **Controlo de Versões:** Utilização de Git e hospedagem de repositórios no GitHub para versionamento de código e trabalho colaborativo.
  * **Deploy & Hospedagem:** Publicação das aplicações web em servidores reais de produção, garantindo links funcionais e acessíveis publicamente.

### 🗓️ UC4: Projeto Integrador de Web Designer
* **Foco:** Unidade transversal que unifica os conhecimentos de todas as UCs em um projeto real.
* **O que foi desenvolvido:**
  * Desenvolvimento em equipa de um website completo, funcional e publicado, partindo de um *briefing* real de mercado ou de uma situação-problema desafiadora.
  * **Avaliação baseada em Evidências:** A nota tradicional é substituída por menções conceituais de competência baseadas na entrega de resultados tangíveis e na evolução documentada através de um **Portfólio Individual**.
  * **Apresentação Profissional (Síntese):** Defesa técnica do projeto simulando uma entrega para um cliente real do mercado de trabalho, justificando as escolhas de arquitetura, semântica do código, acessibilidade e estilização CSS.

## 📋 Critérios Técnicos de Qualidade Aplicados

Todos os códigos contidos neste repositório seguem rigorosamente os padrões de qualidade exigidos pelo mercado e validados institucionalmente:
1. **Semântica Rigorosa:** Cada tag HTML cumpre estritamente o seu papel estrutural.
2. **Separação de Camadas:** Estrutura (HTML), Estilo (Arquivos CSS externos) e Comportamento (Arquivos JS separados) nunca se misturam no mesmo ficheiro.
3. **Responsividade Nativa:** Uso planeado de unidades relativas (`%`, `vw`, `vh`, `rem`, `em`) e Flexbox.
4. **Erros Zero:** Códigos limpos e validados sem alertas nos validadores W3C.
